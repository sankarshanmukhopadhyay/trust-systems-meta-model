# Release v0.9.0

Version: v0.9.0  
Date: 2026-03-14

## Summary

v0.9.0 turns TSMM into an executable meta-model release. The repository now includes a machine-readable graph schema, a reference ecosystem graph, a semantic validator, and reusable graph profiles for common trust-system topologies.

## Added

- `schemas/tsmm-graph.schema.json`
- `examples/tsmm-ecosystem-example.json`
- `examples/profiles/ssi-ecosystem.json`
- `examples/profiles/agent-trust-network.json`
- `examples/profiles/trust-registry-federation.json`
- `examples/profiles/dpi-trust-layer.json`
- `scripts/validate_tsmm_graph.py`
- `docs/model/tsmm-graph-model.md`

## Changed

- updated `scripts/validate_examples.py` to validate graph examples against the new schema
- updated `README.md` to document the executable graph layer and v0.9.0 additions
- updated `docs/index.md` to include graph-model documentation and executable-layer entry points
- bumped repository version metadata to `v0.9.0`

## Why it matters

The core TSMM documents already described the structure of trust systems. This release makes that structure machine-readable.

That creates a practical bridge from concept to tooling:

- implementers can model a trust ecosystem as a constrained graph
- validators can reject structurally invalid trust topologies
- profiles can be shared and reused across ecosystems
- downstream tools can build on a common, portable representation

## Upgrade note

This release is additive. Existing conceptual documents, schemas, and examples remain valid. The graph layer introduces a new executable representation without changing prior core abstractions.
