# TSMM v0.8.0

## Summary

This release introduces TSMM's treatment of dynamic authorization: a framing document that establishes the model's position, a crosswalk mapping XACML 3.0 and ABAC concepts to TSMM abstractions, and a pattern document showing how a fine-grained runtime authorization decision fits within the TSMM governance chain.

The framing document is a stable Tier 1 position statement. The crosswalk and pattern carry `status: experimental`, a new frontmatter convention that signals deliberate deferral of core promotion rather than incomplete or unstable content. The `experimental` status convention is formally defined in this release.

No core abstractions were changed. No schemas were added or modified.

## Added

- `docs/model/dynamic-authorization-framing.md` — stable Tier 1 position statement establishing what dynamic authorization is, how its governing question differs from TSMM's effect-centered question, where the two framings are compatible, where frame drift risk lives, and what implementers should and should not conclude from the crosswalk and pattern documents
- `docs/crosswalks/xacml-abac-crosswalk.md` — experimental crosswalk mapping XACML 3.0 PAP/PDP/PEP/PIP architecture, ABAC concepts, and NGAC to TSMM abstractions; includes a full concept table, PAP/PDP/PEP/PIP-to-TSMM mapping, request/response flow analysis, and treatment of obligations as TSMM conditions
- `docs/patterns/dynamic-authz-pattern.md` — experimental pattern document defining the TSMM-aligned sequence for a dynamic authorization flow; includes structural rules, a sequence diagram, an example scenario in a verifiable credential context, and TSMM core mappings

## Updated

- `docs/documentation-governance.md` — added formal definition of the `status: experimental` frontmatter field and the governance signal it carries; added v0.8.0 note
- `docs/glossary.md` — added Dynamic Authorization, Policy Administration Point (PAP), Policy Decision Point (PDP), Policy Enforcement Point (PEP), and Policy Information Point (PIP) entries
- `docs/extensions/index.md` — added Dynamic Authorization Pattern entry with experimental marker and cross-references to framing and crosswalk documents
- `docs/conformance/tsmm-conformance-checklist.md` — added Dynamic Authorization Pattern extension checklist (DA-1 through DA-7) with experimental marker and framing document reference
- `VERSION` — updated to `v0.8.0`

## Design position

v0.8.0 delivers dynamic authorization as a crosswalk and pattern rather than a schema extension. This was a deliberate architectural decision: introducing PolicyRequest, PolicyResponse, or fine-grained Effect fields as core schema objects risks shifting TSMM's center of gravity from governance conditions toward runtime request evaluation. The crosswalk and pattern approach captures the intellectual value of the dynamic authz framing for TSMM implementers without giving the PDP the conceptual weight that would pull the model off its axis.

The `experimental` tag on the crosswalk and pattern is precise: the framing is understood, the mapping is done, and promotion to core is deferred until implementation experience demonstrates that dynamic authorization concepts are cross-domain, structurally necessary, and stable — per the promotion rule in `docs/extensions/index.md`.

All existing schema/example pairs continue to pass validation. No existing documents required semantic changes.
