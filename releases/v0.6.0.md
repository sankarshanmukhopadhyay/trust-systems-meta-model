# TSMM v0.6.0

## Summary

This release closes the gap between the TSMM conceptual model and practical adoption by adding implementer-readiness tooling, completing the multi-agent governance model, and extending the crosswalk set.

## Added

- `docs/conformance/tsmm-conformance-checklist.md` — structured self-assessment checklist for Minimal, Operational, and Assured profiles, with extension checklists for the Agentic AI, Verifiable Trust Communities, and Assurance extensions
- `docs/getting-started-implementer-guide.md` — opinionated entry points for three implementer types: trust registry operator, verifiable credential issuer/verifier, and agentic system builder
- `docs/patterns/multi-agent-coordination-pattern.md` — governance pattern for chained delegation, sub-delegation permission rules, and oversight mode escalation across agent coordination chains
- `schemas/tsmm-multi-agent-extension.schema.json` — extension schema for multi-agent coordination with support for chained delegations, per-agent actions, effective oversight mode tracking, and coordination-level trace records
- `examples/multi-agent-coordination-instance.json` — worked example of a two-agent procurement workflow with chained delegation and oversight escalation
- `docs/crosswalks/openid-federation-crosswalk.md` — mapping of OpenID Federation 1.0 concepts to TSMM abstractions, covering trust chains, metadata policy, entity statements, and trust marks
- `scripts/check_schema_coverage.py` — validation script that checks whether example instances exercise all properties defined in their paired schemas; surfaces gaps before they become implementer confusion

## Design position

v0.6.0 keeps the core model and extension architecture stable. No core abstractions were changed. The multi-agent coordination extension layers on top of the existing agentic extension rather than modifying it. The conformance checklist and implementer guide are documentation only and do not alter normative schema definitions.
