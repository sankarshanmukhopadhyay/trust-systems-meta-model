# TSMM v0.7.0

## Summary

This release adds the Evidence Artifact model, formalizing how trust systems produce, structure, and consume operational evidence. The addition closes the conceptual gap between rule definition and behavioral proof — the third layer of trust systems that most governance frameworks leave implicit.

## Added

- `docs/model/evidence-artifact.md` — defines the EvidenceArtifact concept as a typed specialization of the core Evidence abstraction, with four artifact types (reconciliation, drift, attestation, conformance), a full property specification, lifecycle expectations, profile mapping, and a promotion rationale
- `schemas/tsmm-evidence-artifact-extension.schema.json` — extension schema for EvidenceArtifact instances, with required fields (`id`, `artifactType`, `producedBy`, `timestamp`) and optional traceability fields (`evidenceHash`, `verificationMethod`, `relatedPolicy`, `relatedAction`, `supportsRefs`)
- `examples/evidence-artifact-instance.json` — worked example demonstrating all four artifact types in a trust registry context, with policy references, supportsRefs to requirements and controls, and verification method annotations

## Updated

- `scripts/validate_examples.py` — added `evidence-artifact-instance.json` / `tsmm-evidence-artifact-extension.schema.json` pair
- `scripts/check_schema_coverage.py` — added same pair to coverage checks
- `docs/extensions/index.md` — added Evidence Artifact Extension reference with schema path
- `docs/glossary.md` — added Evidence Artifact entry with definition, type enumeration, and document reference
- `VERSION` — updated to `v0.7.0`

## Design position

v0.7.0 keeps the core model and all existing extensions stable. No core abstractions were changed, and no existing schemas were modified. The evidence artifact extension layers on top of the existing Evidence entity in the core model rather than altering it.

The EvidenceArtifact concept is positioned as an extension and a core promotion candidate. The promotion decision is deferred until at least one operational implementation demonstrates the model in practice, per the promotion rule in `docs/extensions/index.md`.

The assurance extension's existing `evidenceArtifact` object is structurally consistent with this model. No changes to `tsmm-assurance-extension.schema.json` are required.
