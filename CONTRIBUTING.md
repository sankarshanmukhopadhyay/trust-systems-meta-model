# Contributing

## Scope

TSMM is a reference model repo. Contributions should improve clarity, portability, machine-readability, or cross-ecosystem usefulness.

## Preferred contribution types

- refine abstractions without overfitting to one implementation
- add well-scoped crosswalks to relevant trust-system repositories
- improve glossary precision and terminology consistency
- strengthen examples and schema coverage
- improve documentation freshness, navigation, and GitHub Pages readability

## Contribution rules

- keep the model minimal and operational
- distinguish identity, authority, assurance, conformance, and effect
- avoid introducing repo-specific jargon as universal truth
- update examples and schema when abstractions change
- update `last_reviewed` metadata on touched documents

## Pull request expectations

A solid PR should include:

- the problem being solved
- the abstraction or document changed
- impact on examples, schema, and crosswalks
- any terminology changes that affect other docs

## Versioning

TSMM uses semantic versioning for releases of the reference model and documentation package.
